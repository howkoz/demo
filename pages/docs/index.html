Portal 9094 Demo — Docs

This is the public-safe demo build. All financial figures, counts, and rates are synthetic. Provider metadata (names, markets, methods) is real and unsanitized.

What this is

Portal 9094 is a consolidated payment-operations command center. In production it would pull from the internal data warehouse, DuckDB reconciliation layer, and per-gateway transaction feeds. In this demo, all values are generated by _generate-fake-data.py with a fixed seed so the numbers move like real traffic without leaking anything real.

  • Command Center (index.html) — widget dashboard with ticker, globe, trend, recon, chargeback health, retry ladder, volume-by-market, activity feed, provider directory
  • Provider Directory — searchable table of 38 payment providers with status/region/methods/markets
  • Backend Schema — see SCHEMA.md in the root for the full backend architecture this UI is designed against

Data shape

Each JSON in data/ corresponds to a DB view in the backend spec:

  • markets.json — dimension table (market code, name, region, lat/lon, base approval rate)
  • approval-rates.json — daily fact rows per market: orders, approved, declined, volume_usd, chargebacks, refunds, rate
  • gateways.json — rollup by gateway (NMI, Worldpay, Trust, Nuvei-CA, Braintree, Truemed)
  • recon.json — monthly statement-vs-gateway reconciliation coverage
  • events.json — ops feed (INFO / WARN / OK / FIX)
  • providers.json — provider directory

Filtering

The command center exposes two filter dimensions:

  • Range — 1Y / 6M / 3M / 1M / 2W / 1W. All widgets recompute from the filtered window.
  • Market — "All markets" aggregates by order count; picking a single market filters the fact rows to that market only.

The Window label under the filters shows the actual date range being plotted.

Widgets

  • Global Distribution — Three.js wireframe globe. 4 hub dots (US / NL / SG / TH) and market dots positioned by lat/lon.
  • Approval Rate Trend — ApexCharts area, weighted average by order volume across selected markets.
  • Reconciliation Coverage — radial gauge. % of gateway rows that match a statement row in the latest month.
  • Chargeback Health — line chart in bps of volume, with 100bps card-brand threshold annotation.
  • Retry Ladder — stacked bar showing declined → retried → recovered → unretried for the window.
  • Volume by Market — bar chart of USD volume per market, sorted descending.
  • Activity Feed — recent ops events (INFO / WARN / OK / FIX tags).
  • Provider Directory — inline fuzzy-search table. Full directory lives at Provider Directory.

Layout

Built on Gridstack.js. Widgets are draggable and resizable; charts re-render on resize. Click Reset Layout in the filter bar to reload defaults.

Deployment

This demo is fully static:

python -m http.server 9094

Or deploy the folder to GitHub Pages. No server, no database, no secrets.
